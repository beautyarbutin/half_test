# 产品方案（候选B）：聚焦管理与向导分发的企知助手

## 1. 核心设计理念
区别于纯粹的对话型机器人，候选B方案侧重于**“以知识库管理视角为核心，向导式检索为辅”**的产品控制论。该方案强调知识库的清晰组织、严格隔离、进度可控以及对终端用户的引导，使知识助手不仅是被动回答问题的工具，更是“权限严明、可溯源闭环”的企业知识中枢。

## 2. 用户角色与核心场景

### 2.1 知识管理专家 (Knowledge Admin)
- **核心场景**：掌控及维护知识架构、执行文档的精细化管治（企业制度、业务规范）。
- **流程亮点**：
  - **结构化录入**：配置带有元数据标签的知识库分区（如按部门维度划分），拖拽式上传多类型文档。
  - **切片与索引全透明**：系统不再把解析与重建工作黑盒化，而是提供“解析队列”、“失败文档修复”、“重建索引进度”等监控视图，允许管理员随时干预和全量重做倒排/向量索引。
  - **权限边界与授权**：将具体的知识库精准对收到对应的部门组或业务标签群里。

### 2.2 内部业务员工 (End User)
- **核心场景**：高效定向寻找规章制度、提纲挈领或直接答疑解惑。
- **流程亮点**：
  - **引导式提问边界**：进入问答交互时，系统会优先展示其被授权访问的“知识库列表”（如：“IT部FAQ库”、“2023行政制度库”），鼓励用户在特定范围下提问，从而杜绝由于跨库语义相关导致的回答混淆。
  - **白盒溯源体验**：当助手生成答案时，回答侧边会强制显示详细的上下文卡片（不仅是给出网址链接，还能预览原文高亮片段）。
  - **直达反馈闭环**：如果引用明显答非所问，用户可以点击卡片上的【报错反馈】，这条反馈会直接流转至知识管理专家面板的待办事项中。

## 3. 功能边界与 MVP 里程碑对齐
此产品候选方案完全尊重设定条件：
- **包含**：企业内部知识库问答；结构化知识库上传、更新、删除与人工重建；按“知识库/标签/权限”做的底层 Metadata 隔离过滤；必须支持详细引用出处。
- **排除且首期不需要的**：任何关于模型精调（SFT/RLHF）的页面，任何发往OA及钉钉的复杂流程审批页面卡片交互，以及任何多语言/海外化支持选项。
